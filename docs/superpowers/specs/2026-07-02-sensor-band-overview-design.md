# 传感器波段识别与内部概览设计

## 目标

- 为 `data/test` 中全部 GeoTIFF 原地构建内部 overviews，不依赖 ENVI `.enp` 旁车文件。
- 上传或检查 `GF01`、`LAD08/LAD09`、`SHB02` 文件时自动返回可解释的波段名称与中心波长。
- 保留显式影像元数据最高优先级，避免文件名规则覆盖可信描述或波长。

## 方案

采用后端元数据驱动方案。`inspect_raster()` 先读取 GeoTIFF 自带的描述、标签和波长；仅当这些信息缺失时，才按文件名前缀和波段数选择传感器配置，并为各波段补充标准名称和中心波长。前端沿用现有“波长优先、描述其次、常见顺序兜底”的映射流程。

识别配置：

- `GF01`，4 波段：Blue、Green、Red、NIR。
- `LAD08/LAD09`，7 波段：Coastal Aerosol、Blue、Green、Red、NIR、SWIR1、SWIR2。
- `SHB02`，4 波段：Blue、Green、Red、NIR，对应 Sentinel-2 的 B2、B3、B4、B8。

本项目实际预处理链依据：

- `E:\课设\GF_final\GF01_130200_202302.hdr` 记录 4 波段中心波长为
  `485/555/660/830 nm`。
- `E:\课设\Landsat8原始数据` 的 Landsat 8/9 场景均包含 B1 至 B7，
  `E:\课设\yz\yz.hdr` 记录中间反射率栈为 7 波段、15 m。
- `E:\课设\SHB02原始数据` 的产品 XML 和影像文件确认预处理使用
  10 m 的 B02、B03、B04、B08。

## 边界与错误处理

- 文件名前缀不匹配或波段数不符时不套用传感器配置，继续使用原有兜底。
- Landsat 第 1 波段仅作为 Coastal Aerosol 元数据返回，不映射到平台逻辑蓝光。
- overview 仅用于缩略读取和地图浏览，不改变原始全分辨率像元计算。
- 原地写入前保留当前文件，不删除 `.enp`；`.enp` 仅不再作为平台依赖。

## 验证

- 单元测试覆盖四类前缀、波段数保护、已有描述优先和 Landsat 逻辑映射。
- 运行 Ruff、后端 pytest 和前端构建。
- 对 `data/test/*.tif` 执行 overview 构建，逐个回读层级、文件大小和传感器元数据。
