# 前端地图与指数库修复

## 任务范围

- 修复前端字号过小、遥感工作台布局不合理的问题。
- 增加地图内图层控制：天地图矢量、影像、地形三类底图切换。
- 增加计算前、计算后、对比模式入口。
- 让导入影像有可用经纬度 metadata 时显示空间范围；缺少可叠加 metadata 时不强行显示底图。
- 修复指数库因 Vite 代理旧端口导致无法加载的问题。

## 关键假设

- 当前后端按仓库说明运行在 `http://127.0.0.1:8011`。
- 上传 GeoTIFF 目前没有后端预览 PNG 或瓦片接口，前端只能显示 metadata 范围，不能直接把 GeoTIFF 像素作为浏览器图片图层渲染。
- 任务未自动提交 git commit；当前仓库没有任何提交，也没有 `main` 引用。

## 变更摘要

- `frontend/src/components/MapWorkspace.vue`：重构地图图层状态，增加底图切换、显示模式、结果透明度、导入影像范围提示。
- `frontend/src/App.vue`：向地图组件传入当前导入资产。
- `frontend/src/components/IndexCatalog.vue`：改造指数库卡片，展示公式、波段、范围与限制，提升可读性。
- `frontend/src/components/AppToolbar.vue`、`AssetToolbar.vue`、`AgentDrawer.vue`、`AppStatusBar.vue`、`JobProgressPanel.vue`、`StatisticsDashboard.vue`、`frontend/src/assets/main.css`：提升主要文本字号并减少小字密度。
- `frontend/vite.config.js`：将实际生效的 Vite 代理从旧的 `localhost:8000` 改为默认 `127.0.0.1:8011`，保留 `VITE_API_TARGET` 覆盖能力。

## 验证结果

- `npm run build` 通过。
- 注释扫描 `rg "<!--|// TODO|/\*|无用|临时" frontend/src frontend/vite.config.js` 无命中。
- 通过 `http://127.0.0.1:5174/api/indices` 验证前端代理返回 200。
- 浏览器快照确认状态栏显示 `API 在线` 与 `指数库 30 个`。

## 已知限制

- Vite 构建仍提示 MapLibre 与 ECharts 相关 chunk 超过 500 kB，属于现有依赖体积警告，不阻断构建。
- 真正的“计算前影像像素叠加”需要后端为上传影像生成 PNG 预览或瓦片服务；本次前端已预留对比模式和空间范围显示。
