# pygeoapi 文档服务支持

## 背景

- 用户明确需要打开 pygeoapi 文档，而不是只访问 FastAPI 的 `/docs` 或本地前端。
- 当前本地开发默认端口为后端 `8011`、前端 `5174`；Docker 网关 `8080` 不适用于本地默认启动说明。
- 仓库已有 `backend/app/pygeoapi_processor.py` 与 `infra/pygeoapi/config.yml`，但缺少一条可直接启动 pygeoapi 原生文档服务的项目命令。

## 处理

- 新增 `backend/scripts/start_pygeoapi.py`：
  - 自动基于 `infra/pygeoapi/config.yml` 生成运行期配置到 `backend/data/pygeoapi/config.yml`。
  - 自动生成 `backend/data/pygeoapi/openapi.yml`。
  - 设置 `PYGEOAPI_CONFIG`、`PYGEOAPI_OPENAPI` 与 `PYTHONPATH`。
  - 默认在 `http://127.0.0.1:5000` 启动 pygeoapi 原生服务。
  - 支持 `--port`、`--host`、`--url` 与 `--generate-only`。
- pygeoapi 0.21.0 的 `pygeoapi serve` 内部固定 `debug=True`，会在 Windows 下派生 reloader 子进程；脚本改为导入 `pygeoapi.flask_app.APP` 并显式 `use_reloader=False`，避免验证或用户停止服务后残留端口占用。
- 新增 `backend/app/services/pygeoapi_runtime.py`，把 pygeoapi 配置生成、OpenAPI 生成、环境变量设置和 Flask APP 加载收口到同一处。
- 更新 `backend/app/main.py`，后端启动时自动准备 pygeoapi 文档，并通过 WSGI 挂载到 `/pygeoapi`。
- 更新 `backend/app/api/routes.py`，对 `/processes` 做浏览器与程序化调用分流：
  - `Accept: application/json` 或普通测试调用继续返回现有 35 指数 JSON。
  - `Accept: text/html` 的浏览器访问重定向到 `/pygeoapi/processes`。
- 新增后端根路径 `/openapi`，重定向到 `/pygeoapi/openapi`。
- 更新 `README.md`，补充后端 `8011` 下的 pygeoapi 文档入口。

## 验证

- `D:\miniconda\envs\giskeshe\python.exe scripts\start_pygeoapi.py --generate-only` 通过，成功生成运行期配置和 OpenAPI 文档。
- 临时启动 `scripts\start_pygeoapi.py` 后验证：
  - `http://127.0.0.1:5000` 返回 200。
  - `http://127.0.0.1:5000/openapi` 返回 200。
  - `http://127.0.0.1:5000/processes` 返回 200。
  - 使用 `Accept: text/html` 访问上述三个入口均返回 `text/html`，可作为浏览器文档页打开。
  - 停止验证进程后 `5000` 端口已释放。
- 由于本机 `8011` 已有监听进程，使用临时端口 `8022` 启动同一个 FastAPI 应用验证后端挂载：
  - `GET /processes` + `Accept: application/json` 返回 200 `application/json`。
  - `GET /processes` + `Accept: text/html` 跟随重定向后返回 200 `text/html`。
  - `GET /openapi` + `Accept: text/html` 跟随重定向后返回 200 `text/html`。
  - `GET /pygeoapi` + `Accept: text/html` 返回 200 `text/html`。
- `D:\miniconda\envs\giskeshe\python.exe -m py_compile scripts\start_pygeoapi.py` 通过。
- `D:\miniconda\envs\giskeshe\python.exe -m ruff check app\main.py app\api\routes.py app\services\pygeoapi_runtime.py scripts\start_pygeoapi.py` 通过。
- `D:\miniconda\envs\giskeshe\python.exe -m pytest tests\test_api.py::test_ogc_process_catalog_contains_core_and_legacy_processes -q` 通过，确认程序化 `/processes` 仍返回 35 个指数。

## 风险与后续

- 当前使用 Starlette 内置 `WSGIMiddleware` 挂载 Flask APP；运行正常，但 Starlette 提示该中间件未来会弃用。当前环境未安装 `a2wsgi`，本轮不新增依赖，后续升级 Starlette 时可评估切换。
- `8011` 上若已有旧后端进程，需要重启后才会看到 `/openapi` 和 `/pygeoapi` 新入口。
- Docker Compose 仍未包含独立 pygeoapi 容器；当前容器后端启动后也会自动挂载 pygeoapi 文档入口。
