# 20260623-1104 Agent执行单与知识库导入

## 前置说明

- 用户要求继续完善 Agent：
  - 把确认执行做成更细的可编辑执行单，支持一键改指数、引擎、block size 后提交。
  - 给 RAG 增加外部知识库导入入口，上传指数说明文档后写入 PostgreSQL/知识库并参与检索。
- 本轮继续遵循 `skills/vegetation-agent-designer/SKILL.md`：LLM 不直接提交任务，确认执行必须经过白名单校验。

## 变更内容

- 后端新增 `backend/app/services/agent_knowledge_store.py`：
  - 自动创建 `vegetation_agent_knowledge_documents`。
  - 有 `VIP_DATABASE_URL` 时写 PostgreSQL；不可用时降级内存。
  - 提供 `save_knowledge_document`、`search_persisted_knowledge`。
- RAG 接入持久知识库：
  - `search_index_knowledge` 现在合并内置指数库、请求级外部文档、持久知识库。
  - 补充中文检索词：黄化、氮素、设施农业、病虫害、灌溉等。
- API 扩展：
  - `POST /api/agent/knowledge` 导入外部知识文档。
  - `ConfirmPlanRequest` 支持 `indices`、`engine`、`blockSize`、`priority`。
  - 后端确认执行时校验指数必须来自当前方案的可执行推荐，禁止绕过缺波段检查。
  - `/api/system/capabilities` 增加 `agentKnowledgeStorage`。
- Agent 会话事件：
  - 确认执行后把最终执行单写入 plan 与会话事件 payload。
- 前端 `AgentDrawer.vue`：
  - 新增外部知识库导入区，支持粘贴文本和读取 `.txt/.md/.csv` 文件内容后写入 RAG。
  - 新增可编辑执行单：可勾选指数、选择 Auto/NumPy/Joblib/Torch、调整 block size 和优先级。
  - 提交按钮在未选择指数时禁用。
- 测试：
  - 覆盖知识导入后进入 RAG。
  - 覆盖执行单拒绝未批准/不可执行指数。
  - 覆盖持久知识文档被 Agent RAG 召回。

## 安全边界

- 导入知识文档只作为 RAG 内容，不会触发命令、文件写入或任务提交。
- 执行单只允许编辑白名单参数；后端重新校验指数集合。
- token/base_url 仍只通过前端弹窗作为本次请求参数，不写入知识库、会话库或仓库。
