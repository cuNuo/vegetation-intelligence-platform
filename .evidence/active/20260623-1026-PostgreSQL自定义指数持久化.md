# 20260623-1026 PostgreSQL自定义指数持久化

## 前置说明

- 用户明确要求使用本机 PostgreSQL，并允许通过 `.env` 配置数据库连接。
- 本轮按用户显式指令写入本地 `backend/.env`，该文件已被 `.gitignore` 忽略；日志中不记录明文密码。
- 目标：让 Agent 新建的自定义植被指数能落库持久化，服务重启后自动恢复。

## 变更

- 新增 `VIP_DATABASE_URL` 配置项。
- 新增 `backend/app/services/custom_index_store.py`：
  - 自动创建 `vegetation_custom_indices` 表；
  - 使用 JSONB 保存 expected_range、categories、recommendation_tags、limitations；
  - 使用参数化 SQL 写入和读取；
  - 数据库不可用时降级内存。
- `agent_tools.register_custom_index` 改为注册后写 PostgreSQL，返回 `storage=postgresql|memory`。
- FastAPI lifespan 启动时调用 `load_persisted_custom_indices()` 自动恢复数据库中的自定义指数。
- capabilities 新增：
  - `totalIndexCount`
  - `customIndexCount`
  - `customIndexStorage`
- 前端 Agent 确认执行后新增任务状态轮询，并在任务成功后自动拉取结果、生成统计建议。
- `.env.example` 增加 `VIP_DATABASE_URL` 示例。

## 数据库操作

- PostgreSQL 连接验证成功：`postgres` 用户，本机 `127.0.0.1:5432`。
- 新建数据库：`vegetation_intelligence`。
- 自动建表：`vegetation_custom_indices`。
- 冒烟验证：
  - 写入临时指数 `pg_smoke_nd`；
  - 查询确认 `storage=postgresql`；
  - 验证后删除临时记录。

## 风险

- 当前数据库连接来自本地 `backend/.env`，换机器需要重新配置。
- 自定义指数表尚未接 Alembic 迁移；当前由应用启动时 `CREATE TABLE IF NOT EXISTS` 自举。
- 后续若要团队协作，应把表结构迁移纳入 Alembic，并避免共享真实本机密码。
