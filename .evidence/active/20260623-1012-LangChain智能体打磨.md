# 20260623-1012 LangChain智能体打磨

## 前置说明

- 任务目标：继续按照任务书打磨植被指数分析智能体，使其支持 LangChain、用户自配置 token/base_url、Anthropic 格式、指数库检索与操作、运行期新增正确指数、一键执行、统计信息建议，以及面向演示的运行过程展示。
- 用户新增反馈：模型配置不要常驻占用 Agent 面板，应做成弹出页面；主界面重点保持对话。
- 假设：演示环境允许网络检索，但网络失败不能阻断本地 RAG、规则引擎和一键执行流程。

## 主要改动

- 后端新增 `app/services/agent_tools.py`：
  - 内置指数库轻量 RAG 检索；
  - 默认网络检索入口；
  - 运行期自定义指数注册，包含 AST 白名单校验、必需波段推断和 NumPy 试算；
  - 统计结果规则解读；
  - 面向展示的 agent 过程步骤。
- 后端 `app/services/agent.py`：
  - 接入 LangChain OpenAI-compatible 与 Anthropic provider；
  - 支持请求体传入 token/base_url/model/provider；
  - 计划生成返回 `trace/processSteps/knowledgeHits/webHits/llmStatus/agentMode`；
  - LLM 缺失或失败时降级为规则引擎；
  - 新增统计结果解释编排。
- 后端 API：
  - `/api/agent/plan` 支持 LLM 配置、网络检索、外部文档、自定义指数；
  - 新增 `/api/agent/interpret-results`；
  - 新增 `/api/indices/custom`；
  - capabilities 的 agentMode 更新为 `langchain+rag+web-search+rules`。
- 前端 Agent 面板：
  - 模型配置移动到弹出层；
  - 首屏保留对话输入、网络/RAG状态摘要、自定义指数开关；
  - 方案卡展示运行过程时间线、检索来源、LLM 状态；
  - 支持按统计结果生成建议。

## 风险与后续

- LangChain 依赖已写入 `backend/pyproject.toml`，当前本机环境已有测试可运行；若部署镜像未重新安装依赖，配置 LLM 时会降级规则引擎。
- 默认网络检索使用公开搜索页面解析，适合演示补充来源，但生产环境建议替换为稳定搜索 API 或项目私有知识库。
- 运行期自定义指数只保存在进程内存，重启后丢失；若需要长期保存，应新增持久化表或配置文件并增加审核流程。
