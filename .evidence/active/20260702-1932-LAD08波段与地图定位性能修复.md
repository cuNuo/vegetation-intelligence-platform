## 任务目标

- 修复上传文件被 UUID 重命名后 `LAD08/LAD09` 传感器配置失效的问题。
- 将自动定位与外部底图、TIF 瓦片加载解耦。
- 减少 MapLibre 初始化和资产同步产生的重复网络请求。

## 已确认根因

- 上传保存路径使用 UUID，`inspect_raster()` 无法再看到原始 `LAD08` 文件名前缀。
- 地图样式未就绪时等待 `idle`，导致定位受全部底图瓦片请求影响。
- 初始样式同时加载矢量、影像、地形及三套注记，Playwright 观察到上百个天地图请求。
- `syncMapLayers()` 多次删除并重建同一 TIF source，可能重复触发瓦片请求。

## 处理过程

- 后端 `inspect_raster()` 增加 `filename_hint`；上传保存仍使用 UUID，但元数据检查传入原始文件名。
- 前端存在可靠波长或描述时从“全部未映射”开始推断，不再残留通用 7 波段兜底。
- Landsat 8/9 最终映射：
  - Blue=2、Green=3、Red=4、NIR=5、SWIR1=6、SWIR2=7。
  - Red Edge 未映射；B1 仅显示为 Coastal Aerosol。
- MapLibre 初始样式不再同时注册三套底图；默认只按需加入影像底图和影像注记。
- 当前选中底图始终可见，切换其他底图时再按需添加并复用。
- 使用独立 `mapReady` 状态表示地图初始 load 完成；新增 source 后不再等待
  `isStyleLoaded()` 或 `idle` 才定位。
- source/product 使用 URL 与 bounds 签名做幂等同步；普通透明度或图层状态变化不再删除重建。
- 上传后先显示 footprint 和 PNG preview；本地 TIF source 就绪后自动切换到 TIF。
- 响应式工具栏按实际网格行数计算高度，避免第二行按钮溢出并被地图状态条拦截。
