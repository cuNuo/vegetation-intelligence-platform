## 任务目标

- 优化公式排版、地图颜色条/比例尺/指北针、Agent 方案卡片边界，并创建远程 GitHub 仓库、配置轻量 CI 后同步代码。

## 背景与输入

- 用户反馈：公式展示过于拥挤；计算结果缺少颜色条；地图缺少比例尺与指北针；内部知识库需要增强正确性；后续又指出比例尺太小、指北针重叠、Agent 方案卡片左右边界太小且割裂。
- 已知约束：植被指数计算仍需使用原始 GeoTIFF；GitHub 同步前需完成本地构建、测试和 Hook 检查。

## 处理过程

### 1. 公式与内部知识库

- 做了什么：公式卡片增加横向公式视窗和原始公式等宽行，避免长公式强行折行；内置指数 RAG 知识由“公式+描述”增强为公式、必需波段、期望范围、用途、推荐场景和限制。
- 为什么这么做：长公式需要可核对且不挤压；Agent 选择指数时需要结构化知识，不能只依赖简短描述。
- 结果：公式可读性更高，内部知识召回信息更完整。

### 2. 地图制图要素

- 做了什么：地图新增结果颜色条、MapLibre 比例尺和自定义指北针；比例尺样式放大；指北针移到缩放控件下方更远位置，避免重叠。
- 为什么这么做：遥感结果必须具备基本制图读图要素；比例尺和指北针不能与缩放控件抢位。
- 结果：地图具备颜色条、比例尺、指北针，且控件层次更清楚。

### 3. Agent 方案卡片

- 做了什么：方案卡片改为完整边框、内边距和轻背景，不再只靠一条顶边分隔。
- 为什么这么做：右侧栏窄宽度下，只有顶边会显得割裂且左右边界太小。
- 结果：方案区域更像一个完整工作块，边界更稳定。

### 4. CI

- 做了什么：新增 `.github/workflows/ci.yml`，包含后端 Python 3.11 `ruff + pytest` 和前端 Node 22 `npm ci + npm run build`。
- 为什么这么做：远程仓库需要最小自动验证，覆盖当前主要交付面。
- 结果：CI 配置已落地，等待推送远程后由 GitHub Actions 执行。

### 5. 思考过程节奏二次优化

- 做了什么：Agent 思考队列改为两段式展示，done 事件先以 running 状态出现，停留后再转为 done；首步延迟约 420ms，后续阶段约 1180ms 逐条推进；running 阶段增加脉冲点和“进行中”标识。
- 为什么这么做：后端 SSE 返回很快时，直接展示全部阶段会显得“假流式”；用户需要看到阶段推进，而不是一次性蹦出结果。
- 结果：思考过程更接近真实工作流观感，当前阶段和完成阶段有明确区别。
