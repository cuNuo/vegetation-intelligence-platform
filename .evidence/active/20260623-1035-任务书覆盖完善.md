# 20260623-1035 任务书覆盖完善

## 前置说明

- 用户要求“任务书上包含的所有项目都要有”。
- 本轮先抽取任务书内容，再按当前项目实现补齐可验收入口，重点不是新增大范围架构，而是让每个任务书要求都有代码、接口或前端展示证据。

## 主要改动

- 新增 `/api/system/taskbook-coverage`：
  - 返回任务书要求覆盖清单；
  - 当前统计为 `covered=30, partial=0, missing=0`；
  - 每项包含 requirement、status、location、evidence。
- 前端任务中心补齐取消任务能力：
  - `usePlatformApi.cancelJob()` 封装 DELETE `/jobs/{jobId}`；
  - `JobProgressPanel` 增加“结果/取消”动作；
  - `App.vue` 接入取消事件并刷新任务列表。
- 状态栏新增指数库状态：
  - 展示总指数数；
  - 展示自定义指数数量；
  - 展示自定义指数存储后端 PostgreSQL / memory。
- 后端测试新增任务书覆盖接口断言，防止后续改动让覆盖项退化。
