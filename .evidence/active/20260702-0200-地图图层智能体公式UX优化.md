# 地图图层、智能体对话与公式展示 UX 优化

## 前置说明

- 时间：2026-07-02
- 任务来源：用户反馈地图导入后不自动定位、计算前后不追踪定位、缺少图层控制、智能体对话体验弱、知识库占用主界面、公式展示仍不美观、重复限制文案过多。
- 主要约束：
  - Vue 组件继续使用 `<script setup lang="ts">` 和 Composition API。
  - 地图行为以真实 MapLibre 图层状态为准，不能只用提示文字解释。
  - 不使用 `v-html` 渲染公式，避免不受控 HTML。
  - 不提交 `.env`、测试影像、构建产物或运行中的服务日志。

## 变更记录

- `MapWorkspace.vue`
  - 增加 ArcGIS 风格左侧图层面板，支持底图、导入影像、计算结果、范围框显隐。
  - 支持“只看底图”和“恢复分析图层”。
  - 底图支持矢量、影像、地形切换。
  - 导入影像后自动定位；计算前、计算后、对比按钮会同步追踪到对应范围。
  - 对小范围影像使用显式 `easeTo(center, zoom=14)`，避免 `fitBounds` 在超小范围或图像源刚添加时不稳定。
  - 删除“上传影像像素预览需要后端生成 PNG 或瓦片”等冗余提示。
- `AgentDrawer.vue`
  - 主体改为对话优先：聊天流、状态、输入框成为主交互。
  - 用户发送后先写入本地对话流，再等待后端方案，避免“输入后界面不迁移”的空反馈。
  - 外部知识库移入弹窗，主面板只保留“知识库”入口。
  - 方案详情、执行单、来源、运行过程收敛为可展开区域。
  - 输入框默认清空，使用 placeholder 引导；保存最近用户目标供后续统计解读使用。
- `IndexCatalog.vue`
  - 公式展示改为类 LaTeX 分式排版。
  - 只拆最外层除号，避免括号内部表达式被误拆。
  - 移除每张指数卡重复展示的限制语句。
- `App.vue`
  - 移除首屏和页脚重复解释性文案，保留工作台标题和实际操作面板。

## 风险与后续建议

- 当前公式渲染是轻量解析器，适合已有公式注册表；若后续支持任意用户公式，可进一步引入受控数学排版组件或 AST 渲染。
- 天地图在极高 zoom 级别可能无影像，本轮自动定位上限设为 14，兼顾底图可读与影像范围定位。
- 右侧智能体已改为对话优先，但后续若要接近完整 Codex 体验，还应支持流式响应、会话列表和消息级工具调用展开。
