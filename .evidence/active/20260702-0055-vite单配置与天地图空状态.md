# Vite 单配置与天地图空状态

## 背景

- 用户反馈 `npm run dev -- --host 127.0.0.1 --port 5174` 实际输出为 `vite 127.0.0.1 5174`，并回退到 `http://localhost:5173/`。
- 用户要求只保留一个 `vite.config.js`。
- 用户要求未导入栅格时左侧仍显示天地图底图浏览，右侧保持智能体面板。

## 处理

- 将 `frontend/package.json` 的 `dev` 脚本固化为 `vite --host 127.0.0.1 --port 5174`，避免依赖 shell/npm 参数透传。
- 在 `frontend/vite.config.js` 中补充 `server.host = '127.0.0.1'` 与 `strictPort = true`。
- 在 `frontend/vite.config.js` 中设置 `envDir = '..'`，让仓库根目录 `.env` 的 `VITE_TIANDITU_TOKEN` 对前端生效。
- 删除旧的 `frontend/vite.config.ts` 与 `frontend/vite.config.d.ts`。
- 将 `frontend/tsconfig.node.json` 改为检查唯一的 `vite.config.js`。
- 调整 `MapWorkspace.vue`：地图工作区始终显示底图图层，导入影像和计算结果只作为叠加层。
- 当本地未配置 `VITE_TIANDITU_TOKEN` 时，在地图左下角显示配置提示，不再让用户误判为空状态布局故障。

## 验证

- `npm run build` 通过，仅保留 Vite 大 chunk 警告。
- 重启前端后日志显示：
  `vite --host 127.0.0.1 --port 5174`
  `Local: http://127.0.0.1:5174/`
- 浏览器截图：`output/playwright/tianditu-agent-layout-20260702.png`。
- 当前本机仓库没有 `.env`，因此未配置真实天地图 Token；在线天地图瓦片需要通过本地 `.env` 或部署环境注入。

## 风险

- 不能把天地图 Token 回写到源码或 `.env.example`，否则会再次引入明文凭据风险。
- 以后启动前端建议直接执行 `npm run dev`，不要再附加 `-- --host ... --port ...`。
